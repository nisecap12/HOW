HDC 란?

	H	:	핸들
	DC	:	그리기에 필요한 모든 정보를 가진
			데이터 구조체

	DC 내부의 정보 종류

		HPEN, HBRUSH, HFONT.....

		HBITMAP ( 그릴 공간, 캔버스 )


화면 깜박임....

	1. invalidaterect 함수
	 - 무효화영역 강제 발생
	 - WM_PAINT 강제 호출
	 - 매개변수 HWND, RECT, ERASE
	
	* 위 함수의 ERASE 가 TRUE 라서 생기는 현상.

근본적 원인

	그리는 사이사이의 캔버스가 화면에 그대로 노출되어 있기 때문에....

해결책

	1 번만 그리자.....

	다른 HBITMAP 에 그린 이후에.... 이 HBITMAP 을 한번에 그리자...


더블버퍼링 or 백버퍼링

	임시 버퍼를 생성하여, 해당 버퍼에 랜더링 작업을 모두 진행한 이후
	실제 버퍼로 데이터를 일제히 이동시켜 나타내는 방법

	방법

	1. 뒷종이(HBITMAP)
	2. 뒷종이에 다 그린다.
	3. 뒷종이를 앞종이에 옮겨 그린다.

	문제

		HBITMAP 에 직접 그리는 방법이 존재하지 않는다.

	해결

	1. 기존 HDC와 동일한 HDC 를 하나더 만든다.
	2. HBITMAP 을 만든다.
	3. 새로만든 HDC에 새로만든 HBITMAP 을 넣는다.
	4. DC 간 고속복사 함수로 HBITMAP 을 복사한다.
	5. HBITMAP 과 HDC 를 삭제한다.
